# QMT 数据维护策略 (src/common/数据.py)

## 1. 简介
本策略是实盘环境下的**数据基础设施**，独立于交易策略运行。
**核心目标**：解决 QMT 客户端本地数据可能存在的缺失、断层或非官方归档问题，确保交易策略（如三合一、小市值、ETF轮动）在运行时能获取到准确的历史 K 线。

## 2. 运行机制

### 2.1 核心流程
*   **09:00 前置检查 (Pre-flight Check)**
    *   **触发**：每日开盘前（定时任务 `precheck_and_fix`）。
    *   **逻辑**：
        1.  **抽样检查**：对各类股票池（A股、ETF、指数）进行随机抽样（默认200只）。
        2.  **完整性校验**：检查本地是否包含最近 N 天的日线数据。
        3.  **按需补全**：若抽样通过率低（缺失率 > 20%），则触发**分片下载**逻辑，补齐历史数据。
*   **16:00 盘后归档 (Post-market Sync)**
    *   **触发**：每日收盘后（定时任务 `post_market_download`）。
    *   **逻辑**：
        *   强制下载全市场目标标的最近 5 天的日线数据。
        *   **目的**：用服务器端的“官方归档数据”覆盖本地盘中实时合成的临时数据，消除数据漂移。

### 2.2 关键特性
*   **分片下载**：通过 `_chunk_list` 将大批量股票切分为小块（默认400只/组），防止请求超时或卡死客户端。
*   **进度感知**：下载过程中实时打印进度日志。
*   **微信推送**：集成 `Messager` 类，关键异常和执行结果会推送到企业微信。

## 3. 如何新增数据监控

如果你开发了新策略，需要引入新的数据依赖（例如新增了一个板块或指数），请按以下步骤操作：

### 第一步：定义数据池
在 `G` 类或 `init` 函数上方定义新的全局变量。
```python
# src/common/数据.py

# 例如：新增可转债数据池
g.convertible_bond_pool = ["110011.SH", "128001.SZ", ...] 
# 或者通过板块获取
# g.cb_sector = "上证转债"
```

### 第二步：注册前置检查 (Pre-check)
在 `precheck_and_fix(C)` 函数中，添加检查与预取逻辑。
```python

    # ... 现有代码 ...

    # 新增：可转债数据检查

    )
```

## 4. 配置参数说明
| 参数 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `g.chunk_size` | 400 | 单次 API 请求下载的股票数量，过大易超时 |
| `g.sample_size` | 200 | 前置检查时的抽样数量 |
| `g.max_missing_ratio` | 0.2 | 缺失率容忍度（20%），超过此比例触发全量下载 |
| `g.post_trade_days` | 5 | 盘后强制覆盖最近 N 天的数据 |
